Object Exercise 1: Who Heals The Healer?

A lost medical robot was found trudling around near your encampment. People in your community desperately need it. Unfortunately, its software seems to have been hacked and it has a lot of errors. You must fix it!

  1. Using the Chrome dev console, diagnose the problems and attempt to debug the robot's software in index.js. There are two main objects: the robot itself, and an external instruction set. Both need fixing! You can see the errors already in the console - on the right-hand side, you'll see index.js:#, where the # corresponds to the line number where the error is. Pay close attention and be patient! Coding requires a lot of debugging. Hint: there are a total of 4 syntax bugs.
  2. Now the script runs without errors, and the greet method works (test it out by passing a name (string) as the argument to medicalRobot.greet())! But it won't perform its heal method (try passing a bodypart (string) as an argument to medicalRobot.heal()). It seems to be locked to a certain location. Figure out how to unlock it.
  3. Even when you've fixed eR42, eR42 still sometimes makes mistakes (try the heal method a few times on different bodyparts) - it says that the condition is undefined. Why is that? Fix it. Hint: Math.random() produces a number between 0 and 1, not including 1.
  4. Currently the eR42 has a 50% chance of healing the patient - that's pretty bad. Adjust it up to 85%. 100% wouldn't be any fun, would it?
  5. Wait a second... if something is diagnosed as 'missing', can eR42 heal it? I don't think so. Fix the performHeal method of the medicalInstructionSetV14 object by implementing a if conditional that automatically returns false if the condition is 'missing'.
